🛡️ Non-invasive & read-only. This wizard does not make any changes to your computer, does not install software, and does not send your data to any server. All detection happens entirely in your browser.

What is aibox?

aibox helps you run a fully private, local AI assistant on your own Windows PC using Ollama (Windows-native) and OpenWebUI. Your conversations never leave your machine.

This wizard will walk you through four quick steps:

  1. Baseline — the ideal target configuration (this page)
  2. Detect — best-effort browser detection of your current setup
  3. Confirm — you review and edit the detected values
  4. Impact — we explain how your setup compares to the baseline

🎯 Baseline Target Configuration (Level 00)

The following is the recommended baseline — the configuration this wizard was designed around. Don't worry if you don't match it; the wizard will tell you what to expect.

Component Baseline Value Why It Matters
CPU Intel Core Ultra 7 Efficient cores for CPU inference fallback and background tasks
System RAM 32 GB Model weights, OS, and Docker need headroom — 32 GB is comfortable for 14B+ models
GPU NVIDIA GeForce RTX 5060 Ti × 2 Dual GPU enables larger context windows and faster throughput via GPU offloading
VRAM 16 GB per card (32 GB total) VRAM is the key constraint for GPU inference — larger VRAM = larger models run faster
Storage NVMe SSD, 100 GB free Model files are large (4–20 GB each); NVMe reduces load times
NVIDIA Drivers Installed Required for CUDA-accelerated inference in Ollama
Ollama Windows-native Native Windows install; no WSL2 required for Ollama itself
Docker Desktop Installed Used to run the OpenWebUI container (chat frontend)
OpenWebUI Docker container on port 3000 Chat interface; connects to Ollama via host.docker.internal:11434

ℹ️ What this wizard does NOT do

  • Does not install any software
  • Does not modify any system settings
  • Does not collect or store your data anywhere — everything stays in your browser
  • Does not require login or registration
  • Does not communicate with any server during this assessment

Later stages (not implemented here) will provide optional PowerShell scripts you can review before running. All changes will always be explicit and opt-in.

🗺️ What comes next?

This wizard covers Level 00 (baseline spec) and Level 0 (browser detection). Future stages will provide:

  • Level 1 — PowerShell audit script
    A read-only aibox-audit.ps1 you can run in PowerShell to collect exact CPU, RAM, VRAM, disk, driver, and Docker info. Outputs aibox-profile.json.
  • Level 2 — Optional config registration
    Upload your aibox-profile.json to generate customised next-step scripts. Local-only; no account required.
  • Level 3 — Prerequisite scripts
    Opt-in PowerShell scripts to install Docker Desktop, enable required features.
  • Level 4 — Ollama + OpenWebUI install
    One-command install: Windows-native Ollama, pull a baseline model, and run OpenWebUI on http://localhost:3000.

All future scripts will be viewable on GitHub before you run them. No hidden behaviour.
